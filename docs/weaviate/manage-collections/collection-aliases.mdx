Collection aliases allow you to create alternative names for your collections. This is useful for seamless schema migrations, A/B testing, or providing more convenient names for collections. An alias acts as a pointer to a collection - when you query using an alias name, Weaviate automatically routes the request to the target collection.

:::info Version support
Collection aliases are supported in Weaviate `v1.32.0` and higher.
:::

## Use cases

Collection aliases are particularly useful for:

- **Zero-downtime migrations**: Switch between old and new collections instantly by updating an alias
- **A/B testing**: Route traffic between different collection versions for testing
- **Simplified naming**: Create shorter or more intuitive names for complex collection names
- **API stability**: Maintain consistent API endpoints while changing underlying collections

## Best practices

- **Naming conventions**: Use clear, descriptive alias names that indicate their purpose (e.g., `articles_prod`, `users_latest`)
- **Documentation**: Keep track of which aliases point to which collections, especially in production environments
- **Testing**: Always test alias updates in a staging environment before applying to production
- **Cleanup**: Remove unused aliases to avoid confusion
- **Monitoring**: Monitor query performance when switching aliases between collections

## Limitations

- Aliases are name-based redirects only - they don't modify the underlying collection schema or data
- An alias cannot have the same name as an existing collection or another alias
- Deleting a collection does not automatically delete aliases pointing to it
